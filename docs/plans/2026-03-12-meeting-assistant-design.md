# 会议助手 - 设计文档

> 一个手机网页版会议录音转写和纪要生成工具

## 项目概述

### 目标
通过手机浏览器打开网页，实现：
- 实时语音转文字（边说边出字幕）
- 自动生成结构化会议纪要

### 使用场景
- 主要：线下面对面会议
- 次要：线上视频会议
- 录音时长：约 30 分钟/段，可多段录制

### 用户
- 个人使用，不需要上架发布

```
│                      手机浏览器                          │
│                                                         │
│   ┌─────────┐    音频流     ┌─────────────────────┐     │
│   │ 麦克风   │ ──────────▶  │ WebSocket 连接后端   │     │
│   └─────────┘               └──────────┬──────────┘     │
│                                        │                │
│   ┌─────────────────────────────────────────────────┐   │
│   │              实时显示区域                         │   │
│   │  "今天我们讨论三个议题，第一个是..."               │   │
│   └─────────────────────────────────────────────────┘   │
│                                                         │
│   ┌─────────────────────────────────────────────────┐   │
│   │              本地存储 (IndexedDB)                 │   │
│   │              保存录音片段和会议记录                 │   │
│   └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────┐
│                    后端 (Node.js)                        │
│                                                         │
│   浏览器 ←──WebSocket──→ 后端 ←──WebSocket──→ 讯飞API   │
│                           │                             │
│                           └──────HTTP──────→ DeepSeek   │
└─────────────────────────────────────────────────────────┘
```

### 核心流程

1. **实时转写**：点击"开始录音" → 建立 WebSocket → 麦克风采集 → 实时发送 → 讯飞返回文字 → 实时显示
2. **多段录音**：可多次开始/停止，每段独立保存
3. **生成纪要**：选择片段 → 合并文字 → DeepSeek 生成结构化纪要

## 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 + Vite | 轻量、开发快 |

| 前端部署 | Vercel | 免费 |
| 后端部署 | Railway | 免费额度 |

## 数据结构

### 会议 (Meeting)

```typescript
interface Meeting {
  id: string
  title: string
  createdAt: Date
  updatedAt: Date
  segments: Segment[]
  summary?: Summary
}
```

### 录音片段 (Segment)

```typescript
interface Segment {
  id: string
  meetingId: string
  startTime: Date
  duration: number        // 秒
  transcript: string      // 转写文字
  audioBlob?: Blob        // 原始音频（可选保留）
}
```

### 会议纪要 (Summary)

```typescript
interface Summary {
  id: string
  meetingId: string
  createdAt: Date
  fullText: string        // 完整转写合并
  structured: {
    topics: string[]      // 议题
    keyPoints: string[]   // 讨论要点
    decisions: string[]   // 决议
    actionItems: {        // 待办事项
      person: string
      task: string
    }[]
  }
}
```

## API 设计

### 后端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| WebSocket | `/ws/transcribe` | 实时语音转写 |
| POST | `/api/summarize` | 生成会议纪要 |

### WebSocket 消息格式

**客户端 → 服务端**：
```json
{
  "type": "audio",
  "data": "<base64 encoded audio chunk>"
}
```

**服务端 → 客户端**：
```json
{
  "type": "transcript",
  "text": "识别出的文字",
  "isFinal": false
}
```

### 生成纪要接口

**请求**：
```json
POST /api/summarize
{
  "text": "完整的会议转写文字..."
}
```

**响应**：
```json
{
  "structured": {
    "topics": ["议题1", "议题2"],
    "keyPoints": ["要点1", "要点2"],
    "decisions": ["决议1"],
    "actionItems": [
      {"person": "张三", "task": "完成设计"}
    ]
  }
}
```
